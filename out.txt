For a (2, 3)-locating array, the following two sets have an unsolvable conflict:
	Set 663:	{ 13:{ (f0, 0) (f2, 0) (f3, 0) }; 34:{ (f0, 1) (f2, 0) (f3, 0) } }
	Set 1626:	{ 43:{ (f1, 0) (f2, 0) (f3, 0) }; 52:{ (f1, 1) (f2, 0) (f3, 0) } }
Note that there are many many more examples, but this is just one to demonstrate the problem.
What is the problem?
We want size-2 sets of 3-way interactions, but the first two factors can only take on two levels.
Notice that the first set keeps (f2, 0) and (f3, 0) constant,
	so the only difference between the interactions is that f0 is either 0 or 1.
0 and 1 are the only two values f0 could possible take on, so this is as if to say that f0 does not matter.
Then, ρ(A, {{(f0, 0), (f2, 0), (f3, 0)}, {(f0, 1), (f2, 0), (f3, 0)}}) =  ρ(A, {{(f2, 0), (f3, 0)}, {(f2, 0), (f3, 0)}}).
That is, the size-2 set of those 3-way interactions will occur in the same rows
	as the size-2 set of the 2-way interactions derived by taking out the f0 singles.
Now look at the second set.
It also keeps (f2, 0) and (f3, 0) constant, they exact same 2-way interaction held constant in the first set.
Its third singles? (f1, 0) and (f1, 1), *the two values that f1 can possibly take on*.
The same effect occurs, where this basically says that it does not matter what f1 is,
	boiling the ρ down to just ρ(A, {{(f2, 0), (f3, 0)}, {(f2, 0), (f3, 0)}}).
This shows that these two distinct sets will definitely occur on the same set of rows!!
The location property will be impossible to satisfy.

Even a (2, 1)-locating array will be impossible, as the following two sets will exist:
	{ { (f0, 0) }; { (f0, 1) } }
	{ { (f1, 0) }; { (f1, 1) } }
ρ for both of these sets will be the set of all rows in the array.
What we are seeing is that when it is possible to construct a size-d set that contains every level a given factor can take on,
	it renders the contribution of that factor to the ρ of that set void.
This makes the ρ of that set definitely equivalent to the ρ of some size-d set of a smaller interaction size.
IF THERE EXIST TWO OR MORE FACTORS FOR WHICH THIS IS POSSIBLE, THE LOCATION PROPERTY WILL BE IMPOSSIBLE TO SATISFY.
This is because there will definitely be some pair of size-d sets wherein each set's ρ is effectively reduced to the ρ of the same set.
Then, a precondition to check for is that less than d factors have d levels;
	if this is violated, (d, t)-location is impossible for any t.
